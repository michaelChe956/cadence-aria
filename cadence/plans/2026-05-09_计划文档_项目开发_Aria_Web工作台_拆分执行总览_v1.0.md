# Aria Web 工作台拆分执行总览 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `Aria Web 工作台与逐节点交互 Runtime` 总实施计划拆成 6 个可执行子计划，同时保持 `design` v1.0 的范围、API、UI 和验收标准不变。

**Architecture:** 本文是拆分索引，不替代设计文档，也不改变总计划的技术决策。`cadence/designs/2026-05-09_技术方案_Aria_Web工作台与逐节点交互Runtime设计_v1.0.md` 是验收标准；`cadence/plans/2026-05-09_计划文档_项目开发_Aria_Web工作台与逐节点交互Runtime实现计划_v1.0.md` 是覆盖基准；P1-P6 是执行切片。

**Tech Stack:** Rust 1.95、tokio、serde/serde_json、axum、tower-http、React、Vite、TypeScript、TanStack Router、Tailwind CSS、Radix UI、lucide-react、Vitest、Testing Library、Playwright。

## Execution Rules

- [ ] **Step 1: Execute P1 before any API/UI work**

Run P1 and commit every task listed there.

- [ ] **Step 2: Execute P2 after P1 passes**

P2 depends on P1 runtime types and persistence records.

- [ ] **Step 3: Execute P3 after P2 handler contracts exist**

P3 may use mocked fetch responses until P2 runtime server is running.

- [ ] **Step 4: Execute P4 and P5 after P3**

P4 handles browse/read UI. P5 handles execution controls and rollback UI.

- [ ] **Step 5: Execute P6 only after P1-P5**

P6 is the acceptance gate and must not be skipped.
