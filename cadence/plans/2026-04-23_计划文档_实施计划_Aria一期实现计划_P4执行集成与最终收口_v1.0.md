# Aria Phase 1 P4 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立 `N13-N28` 执行、集成与最终收口闭环，并确保 `final_review._aria.coverage_summary` 可以证明一期任务闭合。

**Architecture:** P4 先实现 worktask 注册与执行准备，再接 coding/testing/review/rework，再接 integration / final closure。执行面先在 fake provider 与 fixture repo 下跑通，真实 provider 接入由 P3 的 provider contract 复用。

**Tech Stack:** Rust、git worktree、fake provider、fixture repo、integration tests。

---

## 0. 评审后准入门槛

P4 启动前必须先落实 `cadence/designs/2026-04-24_技术方案_Aria一期评审后实施规格补齐_v1.1.md` 中与执行、集成和收口相关的裁定：

- 第 10 章：`_aria.traceability_refs` 生成算法与 coverage 判定
- 第 11 章：`N13-N28` 的 `nodeSpecificFields`
- 第 12 章：worktree 路径、lease、并发锁
- 第 13 章：candidate commit、integration branch、N23 cherry-pick、rollback
- 第 14 章：approval gate 与 `N26` 阻断规则
- 第 15 章：fixture repo 与 provider stdout fixture

P4 不允许直接把 worktree diff 合进主分支；一期固定使用 candidate commit + integration branch + rollback 记录。

- [ ] **Step 1: 写失败测试，覆盖 final review / summary / closeout**

断言：
- `final_review`
- `final_review._aria.coverage_summary`
- `final_summary`
- `N28 runtime_snapshot`
- `overallDecision=pass` 时路由到 `N27`
- `overallDecision=followup` 且补丁边界清晰时先进入 `X01 approval_gate_open`
- gate 用户确认后才允许进入 `N26 patch_followup_dispatch`
- `overallDecision=followup` 但补丁边界不清或超过阈值时进入 `X08 manual_intervention_hold`
- `overallDecision=fail` 时进入 `X08 manual_intervention_hold`

- [ ] **Step 2: 实现 `N25 final_review`**

要求：
- 汇总 canonical artifact、projection、bundle、traceability
- 生成 coverage summary

- [ ] **Step 3: 实现 `N25` followup / fail 路由**

要求：
- pass 且 `followupNeeded=false`：进入 `N27`
- followup 且补丁边界清晰、`patch_round_counter` 未超过阈值：进入 `X01 approval_gate_open`
- followup 但补丁边界不清：进入 `X08 manual_intervention_hold`，reason 为 `followup_scope_unclear`
- followup 但超过阈值：进入 `X08 manual_intervention_hold`，reason 为 `patch_round_limit_exceeded`
- fail：进入 `X08 manual_intervention_hold`，reason 为 `final_review_failed`

- [ ] **Step 4: 实现 `N26 patch_followup_dispatch`**

要求：
- 只有 gate 中存在用户显式确认时才允许进入
- 产出新的 `dispatch_package`
- 写入 `dispatch_package._aria.worktask_routing[]`
- `patch_round_counter += 1`
- 重新回到 `N13`

- [ ] **Step 5: 实现 `N27 final_summary`**

要求：
- 只引用已经存在的验证事实
- 不引入未在 `final_review` 中出现的新结论

- [ ] **Step 6: 实现 `N28 session_closeout`**

要求：
- 写最终 checkpoint
- 标记 session 关闭
