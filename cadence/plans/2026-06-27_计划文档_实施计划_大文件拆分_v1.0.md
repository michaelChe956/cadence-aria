# 大文件拆分实施计划

## 目标

将产品源码与产品测试文件中超过 1000 行的文件拆分到 800 行以内，并保持原有功能与测试语义不变。

## 范围

- 包含：`src/`、`tests/`、`web/src/` 下的 Rust、TypeScript、TSX 产品源码与测试文件。
- 排除：`.claude/`、`.codex/`、`.agents/` 等 skill 或工具脚本，以及 lockfile、构建产物、文档。

## 执行策略

1. 新增行数守卫测试，固定扫描范围与扩展名，先确认当前超限会失败。
2. 前端源码采用邻近小模块拆分：
   - `web/src/api/types.ts` 保留为 re-export 入口，实际类型拆到 `web/src/api/types/`。
   - `workspace-ws-store`、`useWorkspaceWs` 拆出类型、selector、stream、artifact、chat rebuild、message handler 等 helper。
   - 页面和面板组件按 tab、panel、view helper 拆到同目录小文件。
3. 前端测试采用 fixture/helper 拆分，测试主题必要时拆成多个同目录测试文件。
4. Rust integration tests 保留现有 `tests/it_*.rs` 聚合入口，只在对应目录下拆子模块和 support，避免增加测试二进制数量。
5. Rust `src/product/workspace_engine/tests/part_03.rs` 按既有 include 模式拆为多个 part 文件。

## 验证

- 红：运行行数守卫测试，确认当前超限文件被报告。
- 绿：拆分后行数守卫通过。
- 定向验证：
  - 前端：相关 `pnpm test -- <测试文件>`。
  - Rust：相关 `cargo test --locked --lib <过滤名>` 或 integration test 过滤命令。
- 全量门禁：
  - `cargo fmt --check`
  - `cargo clippy --all-targets --all-features --locked -- -D warnings`
  - `cargo test --locked`
  - `pnpm test`
  - `pnpm build`
