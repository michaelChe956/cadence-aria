# Chat 对话框优化 P2：协议扩展与 Bug 修复实施计划修订

## 文档信息

- 基于版本：`cadence/plans/2026-05-26_计划文档_实施计划_Chat对话框优化_P2_协议扩展与Bug修复_v1.0.md`
- 对应修订方案：`cadence/designs/2026-05-26_技术方案_Chat对话框五项体验优化_v1.1.md`
- 本轮范围：choice 协议骨架、前端 choice UI、权限按钮状态修复、后端协议错误返回

## 修订点

1. 本轮不承诺真实 Claude Code / Codex adapter 的 choice 事件完整解析，只补协议类型、兼容 match 分支和 fake/test 链路。
2. `choice_response` 与 `permission_response` 一样绕过普通阶段锁，避免 provider 等待期间被拒。
3. `current_run` 为空时，权限响应和选择响应都返回 `ProtocolError`，不静默丢弃。
4. 前端 store 需要在响应后标记原请求 resolved，并追加独立 response entry。
5. 权限请求组件需要点击后立即禁用按钮，resolved 时显示结果标签。

## 执行步骤

### Phase 1：前端 entry 类型与 store 测试

- 修改 `web/src/state/chat-entries.ts` 增加 `choice_request` / `choice_response`。
- 修改 `web/src/state/workspace-ws-store.ts` 增加 `resolveChoiceRequest`，修正 `resolvePermissionRequest`。
- 测试覆盖请求 resolved 标记和响应 entry 追加。

### Phase 2：前端 hook 与渲染

- 修改 `web/src/hooks/useWorkspaceWs.ts`：
  - 处理 `choice_request` 出站消息。
  - 暴露 `sendChoiceResponse`。
  - 保留权限响应错误消息处理。
- 新建 `web/src/components/chat-workspace/entries/ChoiceRequestEntry.tsx`。
- 新建 `web/src/components/chat-workspace/entries/ChoiceResponseEntry.tsx`。
- 修改 `ChatEntryRenderer.tsx` 注册 choice entry。
- 修改 `ChatWorkspacePage.tsx` 传递 choice 回调。
- 测试覆盖单选、多选、自由输入、resolved 隐藏按钮。

### Phase 3：权限按钮回归

- 修改 `PermissionRequestEntry.tsx` 支持 loading/resolved 展示。
- 新建或更新 `PermissionRequestEntry.test.tsx`：
  - 未 resolved 时显示允许/拒绝按钮。
  - 点击允许后按钮 disabled。
  - resolved 后显示结果，隐藏按钮。

### Phase 4：后端协议类型与 handler

- 修改 `src/web/workspace_ws_types.rs`：
  - 增加 `ChoiceOption`、`ChoiceRequest`、`ChoiceResponse`。
  - 增加 `WsOutMessage::ChoiceRequest` 与 `WsInMessage::ChoiceResponse`。
  - 增加序列化测试。
- 修改 `src/cross_cutting/streaming_provider.rs`：
  - 增加 `ChoiceRequestData`、`ChoiceOptionData`、`ProviderEvent::ChoiceRequest`、`ProviderCommand::ChoiceResponse`。
  - 为现有 provider adapter 补齐 match 分支。
- 修改 `src/web/workspace_ws_handler.rs`：
  - 转发 choice request。
  - 接收 choice response 并发送 provider command。
  - `current_run` 为空时返回 protocol error。
