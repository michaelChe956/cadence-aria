# Aria Web 工作台 P5 执行交互与回退 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现执行控制体验：Codex/Claude Code-like Action Composer、自动执行状态、stop signal、rollback preview、dirty 确认、dropped history 和回退后重跑入口。

**Architecture:** P5 连接 P1 runtime、P2 API 和 P3/P4 UI。provider 节点暂停时使用 ActionComposer；内部节点或自动执行阶段使用 AutoActionStatus；回退通过 RollbackDialog 执行 checkpoint 级恢复。

**Tech Stack:** React、TypeScript、Tailwind CSS、Radix Dialog、lucide-react、Vitest、Testing Library、Rust API DTOs。

---

## Design Coverage

P5 覆盖：

- Claude Code/Codex 节点执行前 Codex-like 输入框
- prompt 可编辑或追加指令，确认后服务使用最终 prompt
- 显示 input summary、canonical input refs、context files、output schema、allowed write scope、forbidden actions、verification commands
- 支持单节点临时 policy override，confirm payload 传回最终 override
- 支持确认执行、停止、回退上一轮
- 自动执行阶段显示当前动作、事件摘要和停止入口
- rollback preview、dirty 勾选、执行回退
- WebRuntime rollback 必须调用 `CheckpointService`，不得停留在只返回成功状态的 fake 实现；回退后要刷新 projection 并暴露 dropped history
- dropped 历史灰显，回退后回到 Action Composer 重跑
