# Aria npx 本地化分发 实现计划 · 总览

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `cadence-aria` 打包成可通过 `npx @cadence-aria/cli` 一键本地运行的软件——预编译二进制 + 平台子包分发、前端嵌入二进制、JS launcher 默认启动 web 并开浏览器、Tag 触发多平台自动发布。

**Architecture:** Rust 二进制 `aria`（前端经 `rust-embed` 编译期嵌入）作为唯一运行时；npm 主包 `@cadence-aria/cli` 只含 JS launcher，按 `os`/`cpu` 通过 optionalDependencies 拉取匹配平台子包中的预编译二进制；GitHub Actions 两阶段（多平台原生构建 → 归集发布 npm + GitHub Release）完成分发。

**Tech Stack:** Rust（edition 2024 / rustc 1.95.0、axum、rust-embed、mime_guess）、Node.js（纯 JS launcher、`node:test`）、pnpm/Vite 前端、GitHub Actions。

**对应设计：** `cadence/designs/2026-05-31_技术方案_Aria_npx本地化分发_v1.3.md`

## 依赖关系图

```
P1 (前端嵌入 + build.rs)  ─┐
                           ├─→ P3 (npm 包 + launcher) ─→ P4 (Actions 发布)
P2 (启动检测 + release)   ─┘
```

- P1 是核心硬伤修复（无 P1，二进制分发后白屏），必须最先。
- P2 与 P1 互不依赖代码，但都改 Rust，建议串行避免冲突。
- P3 依赖 P1（嵌入）+ P2（release profile）产出的「发布形态二进制」才能做 `npm pack` 冒烟。
- P4 复用 P3 的组包脚本与子包结构，最后做。

## 关键约束（所有分册通用）

- **TDD**：先写失败测试 → 验证失败 → 最小实现 → 验证通过 → 提交。
- **Rust 命令**（遵循 `cadence/project-rules/build-test-commands.md`）：
  - `cargo fmt --check`
  - `cargo clippy --all-targets --all-features --locked -- -D warnings`
  - `cargo check --locked`
  - `cargo test --locked`
  - 🔴 **禁止** 给任何 cargo 命令加 `-j 1`（并行度由 `.cargo/config.toml` 的 `jobs=8` 托管）。
  - 定向单测用 `cargo test --locked --lib <过滤名>`（必须带 `--lib`）。
- **前端构建前置**：因 P1 引入 build.rs 硬校验 `web/dist` 存在，**任何 `cargo build`/`cargo test` 前必须先 `pnpm -C web build`**（或保证 `web/dist/index.html` 存在）。
- **频繁提交**：每个 Task 末尾提交，提交信息用中文 + `Co-Authored-By` 尾注。
- **工作目录**：worktree `product-workbench-issue-lifecycle`，分支同名。
- **包名**：主包 `@cadence-aria/cli`；平台子包 `@cadence-aria/cli-{linux-x64,darwin-x64,darwin-arm64}`。
- **license**：MIT（仓库根 `LICENSE`）；repo `github.com/michaelChe956/cadence-aria`。

## 完成定义（DoD）

- P1：嵌入资源单测全绿、`ARIA_WEB_DIST` 覆盖生效、build.rs 在 dist 缺失时编译期失败、现有 e2e 仍绿。
- P2：启动检测单测全绿、release 二进制可构建且体积记录、绑定输出行回归测试绿。
- P3：launcher 单测全绿、本地 `npm pack` + `npx` 冒烟在干净临时目录跑通（启动 web、选端口、`--no-open`、参数透传、执行位保留）。
- P4：workflow 文件通过 lint/语法校验、组包脚本本地可跑、文档写明发布前置条件与版本管理。
