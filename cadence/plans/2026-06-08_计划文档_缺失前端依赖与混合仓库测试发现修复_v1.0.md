# Coding Worktree Frontend Bootstrap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让 Coding Workspace 在新 git worktree 缺少前端依赖时引导 Coder 先安装依赖并重试，同时让自动测试发现覆盖 Aria 这类 Rust + `web/` 前端混合仓库。

**Architecture:** 第一阶段不在 worktree prepare 阶段自动联网安装依赖，避免未经用户确认的耗时或网络副作用；先通过 Coder prompt 的明确诊断规则防止把 `node_modules missing` 误判为 `pnpm` 不可用。测试执行器从“命中第一个项目类型就返回”改为“收集根项目和一层子项目的可运行测试命令”，使 `cargo test` 与 `pnpm -C web test` 都能进入测试闭环。

**Tech Stack:** Rust, Tokio, Cargo integration tests, pnpm/Vitest command discovery.

---

## File Structure

- Modify: `src/product/coding_workspace_engine.rs`
  - 负责生成 Coding Workspace Coder 的 full prompt 与 delta prompt。
  - 新增依赖初始化诊断指导，明确 `node_modules missing` / `tsc EACCES` / `vitest EACCES` 应先检查 `pnpm --version` 并执行 `pnpm -C <package-dir> install --frozen-lockfile`。
- Modify: `src/product/test_executor.rs`
  - 负责发现、推断和执行测试命令。
  - 改为收集多个测试命令，并支持一层子目录 `package.json`，例如 `web/package.json`。
  - 顺手移除 `discover_test_commands` 中禁止的 `cargo test -j 1`。
- Modify: `tests/it_product/product_test_executor.rs`
  - 覆盖混合仓库测试命令发现。
  - 覆盖子目录 package 没有 test script 时不生成 pnpm 测试命令。

## Non-Goals

- 本计划不自动在 `execute_worktree_prepare` 阶段运行 `pnpm install`。原因：这会引入网络、耗时、npm registry、用户内网配置和隐式写入 `node_modules` 的产品语义，适合单独做“worktree dependency bootstrap”功能。
- 本计划不改变 Provider 权限策略，不绕过用户对 Coder 命令执行的审批。

    let worktree_path = worktree_path.as_ref();

```rust
fn package_test_commands(worktree_path: &Path, inferred: bool) -> Vec<TestCommandSpec> {
    let mut specs = Vec::new();
    push_package_test_command(worktree_path, None, inferred, &mut specs);

    let Ok(entries) = std::fs::read_dir(worktree_path) else {
        return specs;
    };
    let mut child_dirs = entries
        .filter_map(Result::ok)
        .filter(|entry| entry.path().is_dir())
        .filter_map(|entry| {
            let name = entry.file_name().to_string_lossy().to_string();
            (!ignored_test_discovery_dir(&name)).then_some((name, entry.path()))
        })
        .collect::<Vec<_>>();
    child_dirs.sort_by(|left, right| left.0.cmp(&right.0));

    for (relative_dir, path) in child_dirs {
        push_package_test_command(&path, Some(&relative_dir), inferred, &mut specs);
    }
    specs
}

fn push_package_test_command(
    package_dir: &Path,
    relative_dir: Option<&str>,
    inferred: bool,
    specs: &mut Vec<TestCommandSpec>,
) {
    if !package_json_has_test_script(&package_dir.join("package.json")) {
        return;
    }
    let prefix = if inferred { "inferred_node" } else { "node" };
    let id = relative_dir
        .map(|dir| format!("{prefix}_{}", sanitize_command_id_fragment(dir)))
        .unwrap_or_else(|| prefix.to_string());
    let command = relative_dir
        .map(|dir| {
            vec![
                "pnpm".to_string(),
                "-C".to_string(),
                dir.to_string(),
                "test".to_string(),
            ]
        })
        .unwrap_or_else(|| vec!["pnpm".to_string(), "test".to_string()]);
    specs.push(TestCommandSpec { id, command });
}

fn ignored_test_discovery_dir(name: &str) -> bool {
    matches!(
        name,
        ".aria" | ".git" | ".worktrees" | "node_modules" | "target" | "dist"
    )
}

fn sanitize_command_id_fragment(value: &str) -> String {
    value
        .chars()
        .map(|ch| {
            if ch.is_ascii_alphanumeric() {
                ch.to_ascii_lowercase()
            } else {
                '_'
            }
        })
        .collect()
}
```
