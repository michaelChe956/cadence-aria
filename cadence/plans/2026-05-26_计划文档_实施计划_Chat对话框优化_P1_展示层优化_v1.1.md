# Chat 对话框优化 P1：展示层优化实施计划修订

## 文档信息

- 基于版本：`cadence/plans/2026-05-26_计划文档_实施计划_Chat对话框优化_P1_展示层优化_v1.0.md`
- 对应修订方案：`cadence/designs/2026-05-26_技术方案_Chat对话框五项体验优化_v1.1.md`
- 本轮范围：消息分组、执行事件折叠、审核者同构展示、完成态 artifact 提取

## 修订点

1. `MessageGroup.id` 使用首 entry id 和序号生成，不能只用 `group:${node_id}`。
2. `permission_response`、`choice_response` 作为 standalone entry，不进入消息组。
3. `permission_request`、`gate_prompt`、`choice_request` 作为 interrupt entry。
4. Artifact 只做 provider 完成态提取，不做流式 artifact pane 增量更新。
5. TDD 顺序调整为先补分组纯函数测试和组件测试，再改渲染入口。
