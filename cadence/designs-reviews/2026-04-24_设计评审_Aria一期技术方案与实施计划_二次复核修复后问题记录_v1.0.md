## 4. 阻断问题

### P0-1. `ProjectionPayload` 仍缺少 `Serialize` 派生

**位置**

`cadence/designs/2026-04-24_技术方案_Aria一期评审后实施规格补齐_v1.1.md`

**问题**

`ArtifactProjectionRecord` 派生了 `Serialize` / `Deserialize`，并包含字段：

```rust
pub payload: ProjectionPayload,
```

但 `ProjectionPayload` 当前仍只派生了 `Deserialize`：

```rust
#[derive(Debug, Clone, PartialEq, Deserialize)]
#[serde(tag = "payloadKind", content = "payload", rename_all = "snake_case")]
pub enum ProjectionPayload {
    SpecProjection(SpecProjection),
    DesignProjection(DesignProjection),
    PlanProjection(PlanProjection),
}
```

**影响**

若研发按补齐规格中的 Rust 蓝图直接实现，`ArtifactProjectionRecord` 的 `Serialize` 派生会失败，P2 projection golden JSON 输出链路无法成立。

**修订要求**

将 `ProjectionPayload` 改为：

```rust
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
#[serde(tag = "payloadKind", content = "payload", rename_all = "snake_case")]
pub enum ProjectionPayload {
    SpecProjection(SpecProjection),
    DesignProjection(DesignProjection),
    PlanProjection(PlanProjection),
}
```

**建议验收**

- 文档中 `ProjectionPayload` 明确派生 `Serialize, Deserialize`。
- P2 实现时增加或保留 projection serialization test，覆盖 `ArtifactProjectionRecord` 序列化为 golden JSON。

## 5. 非阻断建议

### P2-1. `RiskEntry.severity/status` 后续建议收紧为 enum

**问题**

`RiskEntry` 已补齐 canonical 字段，但 `severity`、`status` 当前仍是 `String`。

**影响**

这不会阻断当前文档冻结，但代码实现阶段若继续使用裸 `String`，容易接受非法值，例如未知 severity 或 status。

**建议**

实现阶段建议定义明确枚举：

- `RiskSeverity = high / medium / low`
- `RiskStatus = open / mitigated / accepted / escalated`

并在 JSON schema / Rust serde / fixture 中统一校验。
