# Aria Phase 1 P1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立 Aria 一期的 Rust 工程骨架、REPL <-> daemon wire protocol、transport、daemon session 状态与 checkpoint 基线。

**Architecture:** 先把最底层通信与进程边界定死，再实现最小可联调的 REPL/daemon 骨架。P1 不实现 planning/execution 业务，只实现之后所有 plan 都依赖的协议和运行时壳。

**Tech Stack:** Rust、Cargo、serde、Unix socket / named pipe、JSON、basic fs persistence。

---

## 1. 范围与出口

P1 完成后，必须满足：

1. 仓库具备可编译的 Rust 工程骨架
2. REPL 与 daemon 可以通过正式 wire protocol 建立连接
3. 支持 `hello/attach/subscribe/get_status/detach`
4. daemon 能写最小 checkpoint / runtime snapshot
5. REPL 可接收 event 流并支持 reconnect 基线

不在 P1 范围内：

- provider 调用
- projection / `_aria`
- OpenSpec bundle
- planning / execution 节点业务逻辑
