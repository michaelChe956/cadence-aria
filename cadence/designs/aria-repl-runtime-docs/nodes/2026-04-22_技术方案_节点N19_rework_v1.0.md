# 节点文档：N19 rework

## 1. 节点标识

- 节点 ID：`N19`
- 节点类型：Agent 业务
- 主执行者：Codex
- 所属链路：返工
- 版本：v1.0

## 2. 节点目的

根据 testing 或 code review 的失败结果对当前 WorkTask 做针对性修复，再次返回验证链路。

## 3. 进入条件

- `testing_report.pass_fail_status = fail`
  或
- `code_review_report.review_decision = revise`

## 4. 输入契约

| 输入 | 来源 | 必填 | 最小字段 | 缺失处理 |
|------|------|------|----------|----------|
| failure source report | `N17`/`N18` | 是 | failures/findings | 回退上游 |
| current code state | worktree | 是 | path | 回退 `N14` |

## 5. Aria 驱动动作

1. 组装 rework package。
2. 把失败原因和验收标准交给 Codex。
3. 收集返工结果并写新报告。

## 6. Provider 执行契约

- 角色：`executor`
- 默认 provider：Codex
- 必做动作：
  - 只处理当前失败项
  - 输出修复摘要
  - 不扩大无关改动

## 7. 输出产物

- 更新后的 `coding_report` / `testing_report` / `code_review_report`
- `runtime_snapshot:N19.rework`

## 8. 输出产物最小格式

返工输出至少包含：

- `rework_reason`
- `resolved_items`
- `remaining_items`
- `recommended_next_node`

## 9. 完成判定

返工结果已形成新的上游产物，并可再次进入 testing 或 code_review。

## 10. 失败与回流

- 连续返工超阈值：进入人工介入
- 修复方向不清：挂 gate

## 11. 交接到下一节点

- 一般先回 `N17 testing`
- 纯 review 缺陷可直接回 `N18 code_review`

## 12. 关联横切能力

- `provider_run_lifecycle`
- `artifact_validate`
- `approval_gate`
- `manual_intervention`
