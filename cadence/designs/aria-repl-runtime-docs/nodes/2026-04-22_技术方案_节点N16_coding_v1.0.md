# 节点文档：N16 coding

## 1. 节点标识

- 节点 ID：`N16`
- 节点类型：Agent 业务
- 主执行者：Codex
- 所属链路：编码
- 版本：v1.0

## 2. 节点目的

在独立 worktree 中完成指定 WorkTask 的代码实现。

## 3. 进入条件

- `nextExecutionNode = N16`
- worktree 已就绪

## 4. 输入契约

| 输入 | 来源 | 必填 | 最小字段 | 缺失处理 |
|------|------|------|----------|----------|
| WorkTask payload | `dispatch_package` | 是 | task goal、acceptance | 回退 `N12` |
| design refs | `N07`/`N09` | 是 | design ref | 回退 `N07` |
| worktree ref | `N14` | 是 | path | 回退 `N14` |

## 5. Aria 驱动动作

1. 组装 coding package。
2. 把 worktree、任务边界、验收标准交给 Codex。
3. 收集实现摘要和变更记录。

## 6. Provider 执行契约

- 角色：`executor`
- 默认 provider：Codex
- 必做动作：
  - 在指定 worktree 中编码
  - 不越过任务边界
  - 输出实现摘要

## 7. 输出产物

- `coding_report`
- `runtime_snapshot:N16.coding`

## 8. 输出产物最小格式

`coding_report` 至少包含：

- `workTaskId`
- `changed_files`
- `implementation_summary`
- `known_risks`

## 9. 完成判定

代码改动已落在 worktree，且 `coding_report` 已通过校验。

## 10. 失败与回流

- provider 失败：retry/gate
- 输出越界：进入人工介入

## 11. 交接到下一节点

默认进入 `N17 testing`。

## 12. 关联横切能力

- `provider_run_lifecycle`
- `worktree_lifecycle`
- `artifact_validate`
