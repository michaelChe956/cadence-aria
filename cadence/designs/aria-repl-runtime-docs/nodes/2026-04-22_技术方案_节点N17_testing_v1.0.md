# 节点文档：N17 testing

## 1. 节点标识

- 节点 ID：`N17`
- 节点类型：Agent 业务
- 主执行者：Codex
- 所属链路：测试
- 版本：v1.0

## 2. 节点目的

验证当前 WorkTask 的实现是否满足验收标准和必要测试要求。

## 3. 进入条件

- coding 已完成或 rework 后重新验证

## 4. 输入契约

| 输入 | 来源 | 必填 | 最小字段 | 缺失处理 |
|------|------|------|----------|----------|
| `coding_report` | `N16`/`N19` | 是 | ref | 回退上游 |
| worktree ref | `N14` | 是 | path | 回退 `N14` |
| acceptance checks | `plan`/`dispatch_package` | 是 | checks | 回退 `N11`/`N12` |

## 5. Aria 驱动动作

1. 组装 testing package。
2. 调用 Codex 执行测试或测试补充。
3. 收集测试结果。

## 6. Provider 执行契约

- 角色：`executor`
- 默认 provider：Codex
- 必做动作：
  - 运行必要测试
  - 记录失败原因
  - 给出通过/失败结论

## 7. 输出产物

- `testing_report`
- `runtime_snapshot:N17.testing`

## 8. 输出产物最小格式

`testing_report` 至少包含：

- `tests_run`
- `pass_fail_status`
- `failures`
- `next_recommendation`

## 9. 完成判定

测试结果已明确且可路由到 review 或 rework。

## 10. 失败与回流

- 测试失败：进入 `N19 rework`
- 测试环境异常：gate 或人工介入

## 11. 交接到下一节点

- 通过：进入 `N18 code_review`
- 失败：进入 `N19 rework`

## 12. 关联横切能力

- `provider_run_lifecycle`
- `worktree_lifecycle`
- `artifact_validate`
- `approval_gate`
