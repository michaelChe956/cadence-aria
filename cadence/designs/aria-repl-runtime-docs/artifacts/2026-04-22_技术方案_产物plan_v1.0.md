# 产物规范：plan

## 1. 产物标识
- 类型 ID：`ART-PLAN`
- 类别：文档产物
- 默认节点：`N11`

## 2. 产物目的
定义实施工作包、依赖、验收与并行提示，是 dispatch 的直接输入。

## 3. 产出时机
readiness 通过后产出。

## 4. 产物存储位置
默认写入 `cadence/plans/`。

## 5. 最小字段清单
| 字段 | 必填 |
|------|------|
| `work_packages` | 是 |
| `dependencies` | 是 |
| `acceptance_checks` | 是 |
| `parallelism_hints` | 是 |

## 6. 推荐结构
工作包、依赖、验收、并行说明。

## 7. 固定格式示例
```md
# Plan
## Work Packages
## Dependencies
## Acceptance Checks
## Parallelism Hints
```

## 8. 校验规则

### L1 存在性校验
- `work_packages` 存在且为非空数组（至少 1 个）
- `dependencies` 存在（允许为空数组 `[]`）
- `acceptance_checks` 存在且为非空数组（至少 1 组）
- `parallelism_hints` 存在（允许为空数组 `[]`）

### L2 结构性校验
- `work_packages` 为数组，每个元素为对象（含 `id`、`description`、`estimated_scope` 字段）
- `dependencies` 为数组，每个元素为对象（含 `from`、`to` 字段，值为 work package ID）
- `acceptance_checks` 为数组，每个元素为对象（含 `work_package_id`、`checks` 字段）
- `parallelism_hints` 为数组，每个元素为对象（含 `group_id`、`work_package_ids` 字段）

### L3 语义性校验（二期增强）
- `work_packages` 中每个 work package 应可映射到 design 中的组件
- `dependencies` 不应包含环路

## 9. 交接规则
供 `N12` 消费。

## 10. 版本与修订规则
方案变化时升级版本。

## 11. 失败与缺失处理
缺 work package 则回流 `N11`。
