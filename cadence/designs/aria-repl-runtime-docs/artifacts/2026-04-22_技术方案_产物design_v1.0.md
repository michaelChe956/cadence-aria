# 产物规范：design

## 1. 产物标识
- 类型 ID：`ART-DESIGN`
- 类别：文档产物
- 默认节点：`N07`

## 2. 产物目的
定义实现方案，是后续 design review、plan、coding 的设计真相源。

## 3. 产出时机
spec 通过后产出；设计修订后更新。

## 4. 产物存储位置
默认写入 `cadence/designs/`。

## 5. 最小字段清单
| 字段 | 必填 |
|------|------|
| `architecture_summary` | 是 |
| `frontend_design` | 条件 |
| `backend_design` | 条件 |
| `data_model` | 是 |
| `api_contracts` | 是 |
| `risk_list` | 是 |

> **条件必填规则**：`frontend_design` 和 `backend_design` 至少存在一项。纯后端项目可省略 `frontend_design`，纯前端项目可省略 `backend_design`。

## 6. 推荐结构
架构摘要、前端设计、后端设计、数据模型、API、风险。

## 7. 固定格式示例
```md
# Design
## Architecture Summary
## Frontend Design
## Backend Design
## Data Model
## API Contracts
## Risks
```

## 8. 校验规则

### L1 存在性校验
- `architecture_summary` 存在且非空
- `frontend_design` 或 `backend_design` 至少存在一项
- `data_model` 存在且非空
- `api_contracts` 存在且非空
- `risk_list` 存在（允许为空数组 `[]`）

### L2 结构性校验
- `architecture_summary` 为字符串类型，长度 >= 50 字符
- `frontend_design`（若存在）为字符串类型
- `backend_design`（若存在）为字符串类型
- `data_model` 为字符串或对象类型
- `api_contracts` 为字符串或数组类型
- `risk_list` 为数组，每个元素为 Risk Registry 中的 `riskId` 引用

### L3 语义性校验（二期增强）
- `architecture_summary` 应包含组件关系描述
- `risk_list` 中每个风险应在 Risk Registry 中有对应条目

## 9. 交接规则
供 `N08`、`N10`、`N11`、`N25` 消费。

## 10. 版本与修订规则
重大修订升主版本，小修订升小版本。

## 11. 失败与缺失处理
缺章节回流 `N07`。
