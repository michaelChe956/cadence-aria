# 产物规范：design

## 1. 产物标识
- 类型 ID：`ART-DESIGN`
- 类别：文档产物
- 默认节点：`N07`

## 2. 产物目的
定义实现方案，是后续 design review、plan、coding 的设计真相源。

## 3. 产出时机
spec 通过后产出；设计修订后更新。

## 4. 产物存储位置
默认写入 `cadence/designs/`。

## 5. 最小字段清单
| 字段 | 必填 |
|------|------|
| `architecture_summary` | 是 |
| `frontend_design` | 是 |
| `backend_design` | 是 |
| `data_model` | 是 |
| `api_contracts` | 是 |
| `risk_list` | 是 |

## 6. 推荐结构
架构摘要、前端设计、后端设计、数据模型、API、风险。

## 7. 固定格式示例
```md
# Design
## Architecture Summary
## Frontend Design
## Backend Design
## Data Model
## API Contracts
## Risks
```

## 8. 校验规则
六个章节必须齐全。

## 9. 交接规则
供 `N08`、`N10`、`N11`、`N25` 消费。

## 10. 版本与修订规则
重大修订升主版本，小修订升小版本。

## 11. 失败与缺失处理
缺章节回流 `N07`。
