# 产物规范：spec

## 1. 产物标识
- 类型 ID：`ART-SPEC`
- 类别：文档产物
- 默认节点：`N05`

## 2. 产物目的
作为设计阶段的需求真相源，明确边界、用户故事、功能需求和成功标准。

## 3. 产出时机
澄清完成后产出；需求边界变化时修订。

## 4. 产物存储位置
默认写入 `cadence/prds/`。

## 5. 最小字段清单
| 字段 | 必填 |
|------|------|
| `scope` | 是 |
| `user_stories` | 是 |
| `functional_requirements` | 是 |
| `success_criteria` | 是 |
| `open_items` | 是 |

## 6. 推荐结构
范围、用户故事、功能需求、成功标准、待确认项。

## 7. 固定格式示例
```md
# Spec
## Scope
## User Stories
## Functional Requirements
## Success Criteria
## Open Items
```

## 8. 校验规则
五个主章节必须存在，`functional_requirements` 至少 1 项。

## 9. 交接规则
供 `N06`、`N07`、`N10`、`N25` 消费。

## 10. 版本与修订规则
范围变化升版本；小修订升小版本。

## 11. 失败与缺失处理
缺章节回流 `N05`。
