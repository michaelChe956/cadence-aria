# Chat 对话框五项体验优化设计评审修订

## 文档信息

- 基于版本：`cadence/designs/2026-05-26_技术方案_Chat对话框五项体验优化_v1.0.md`
- 修订目的：修正 v1.0 中会导致实现偏差、协议误判或范围膨胀的点
- 执行原则：不覆盖 v1.0；本轮按本文约束执行，未列出的设计仍继承 v1.0

## 评审结论

v1.0 的五项体验优化方向成立，但需要收敛边界并修正以下问题：

1. 消息分组需要稳定且唯一的 group id，不能只使用 `node_id`。
2. `permission_response` 与 `choice_response` 是用户响应记录，必须独立渲染，不进入 provider 消息组。
3. `permission_request`、`gate_prompt`、`choice_request` 是组内 interrupt，可作为当前 provider 消息组中的交互卡片。
4. `choice_request` / `choice_response` 本轮先建立协议骨架、store、hook、UI 与 fake/test 链路；真实 Claude Code / Codex adapter 的 choice 事件解析需后续基于真实事件格式调研。
5. 权限确认按钮修复需要覆盖前端 resolved/loading 状态和后端 `current_run` 缺失错误返回，不能静默丢弃。
6. Artifact 提取本轮只做完成态提取：优先 `<artifact>...</artifact>`，缺失结束标签时取开始标签后内容，否则回退到第一个 Markdown 一级标题；暂不做流式 artifact pane 增量更新。

## 修订后的消息分组规则

`groupEntries(entries)` 只在渲染层工作，不改变 store 数据结构。

- `provider_stream`：作为组内 primary entry。
- `execution_event`：归入当前同 `node_id` 组的 inline events，默认折叠。
- `permission_request`、`gate_prompt`、`choice_request`：归入当前同 `node_id` 组的 interrupt entries。
- `permission_response`、`choice_response`、`stage_change`、`artifact_update`、`review_verdict`、`error`：作为 standalone entry。
- 当 standalone entry 切分同一个 `node_id` 的连续片段时，后续 group 必须生成新的稳定 id。
- group id 使用首个组内 entry id 加序号兜底，避免 `node_id` 重复造成 React key 冲突。

## 修订后的权限/选择响应规则

- 点击权限按钮后，前端立即进入 loading/resolved 状态，禁用按钮，避免重复响应。
- 后端收到 `permission_response` 时，如果 `current_run` 为空，返回 `protocol_error`，不再静默丢弃。
- `choice_response` 与 `permission_response` 一样属于 provider 等待期间的用户输入，WebSocket handler 不应被普通阶段锁拒绝。
- `choice_response` 需要在聊天列表追加独立响应记录，便于用户回看自己的选择。

## 修订后的 Artifact 范围

本轮只处理 provider 完成后的 artifact markdown：

- 存在完整 `<artifact>...</artifact>`：提取标签内内容并 trim。
- 只有 `<artifact>` 开始标签：提取开始标签后的内容并 trim。
- 没有 artifact 标签但存在 Markdown 一级标题：从第一个 `# ` 标题开始截取。
- 以上都不满足：保留 trim 后原文。

流式 chunk 中跨片段识别 `<artifact>`、实时更新 artifact pane 属于后续增强，不纳入本轮验收。

## 本轮验收标准

1. 作者/审核者 provider stream 与执行事件按 `node_id` 合并为视觉消息组，执行事件默认折叠且可展开。
2. 权限请求和选择请求能作为组内交互卡片渲染，响应记录独立显示。
3. 权限按钮点击后状态立即变化，重复点击被禁用；后端缺失运行上下文时返回协议错误。
4. 前端可渲染单选、多选和自由输入 choice request，并发送 choice response。
5. 后端完成态 artifact 只保留结构化产物内容。
