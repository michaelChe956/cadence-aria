# state.json phase 状态未同步问题

## 现象

E2E 流程已实际推进到 N08 design review 阶段，但 `state.json` 的 `phase` 字段始终停留在 `"intake"`，未随节点推进而更新。

## 证据

### state.json（当前）

````json
{
    "change_id": "aria-fibonacci-square",
    "openspec_bootstrap_status": "bootstrapped",
    "phase": "intake",
    "task_id": "task_0001"
}
````

### node-events.jsonl（已推进到 N08）

| 时间 (UTC) | 事件 | 节点 |
|-----------|------|------|
| 18:10:02 | node_enter | N04 |
| 18:11:15 | node_exit (completed, 73s) | N04 |
| 18:11:15 | node_enter | N05 |
| 18:11:49 | node_exit (completed, 34s) | N05 |
| 18:11:49 | node_enter | N06 |
| 18:12:21 | node_exit (completed, 31s) | N06 |
| 18:12:21 | node_enter | N07 |
| 18:14:43 | node_exit (completed, 141s) | N07 |
| 18:14:43 | node_enter | N08 |

## 影响

1. **外部监控失效**：依赖 `state.json` 的监控脚本（包括本轮的 Monitor）无法准确反映当前阶段
2. **调试困难**：无法通过 state.json 快速判断 E2E 实际进展
3. **可能隐藏更深的状态管理问题**：如果 phase 不更新，其他状态字段是否也存在同步问题？

## 待调查方向

1. `state.json` 的写入逻辑位于哪个模块？是否只在特定节点（如 intake 完成时）写入一次？
2. 节点状态转换时是否有更新 `phase` 的代码路径？是否被遗漏？
3. 是否存在多个 state 来源（如内存中的 state 与文件中的 state）不同步？

## 下一步

- 待本轮 E2E 完成后，在源码中搜索 `state.json` 的写入位置
- 检查节点推进时是否调用了 state 更新方法
