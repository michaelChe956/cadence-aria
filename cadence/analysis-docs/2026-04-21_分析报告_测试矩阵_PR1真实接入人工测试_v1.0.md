## 2. 通用检查规则

### 2.1 路径约定

任务目录：

```text
cadence/cache/aria/tasks/<task-id>/
```

关键文件：

```text
cadence/cache/aria/tasks/<task-id>/state.yaml
cadence/cache/aria/tasks/<task-id>/artifacts/
```

### 2.2 判定优先级

1. `state.yaml`：状态事实源
2. artifacts：正式证据源
3. 命令输出：辅助观察面

### 2.3 常用命令

```bash
pnpm build
node dist/src/index.js aria:doctor
node dist/src/index.js aria:intake "<title>"
node dist/src/index.js aria:start --task-id <task-id>
node dist/src/index.js confirm-spec --task-id <task-id>
node dist/src/index.js confirm-plan --task-id <task-id>
node dist/src/index.js aria:run --task-id <task-id>
node dist/src/index.js aria:status --task-id <task-id>
node dist/src/index.js aria:result --task-id <task-id>
node dist/src/index.js aria:retry --task-id <task-id>
node dist/src/index.js aria:cancel --task-id <task-id>
scripts/verify-real-integration.sh --task-id <task-id>
```

### 7.3 其他受控异常

若需稳定复现 `mismatch-task`、`needs_patch`、`review failed`，建议参考：

- `tests/fixtures/fake-binaries.ts`

对应思路：

- 包装脚本根据 prompt 输出合法 YAML

- 保证 `producer`、`source_capabilities` 等其余字段完整，这样才能只验证单一故障点
